Inheritance

Classes can be organized into a hierarchy. Classes in the lower hierarchy inherit all the variables and methods from higher hierarchies. This makes sure that redundant variables and methods are only written once in the superclass and do not need to be repeated in all subclasses. 


Syntax: the keyword "extends" is used to show inheritance

class SubclassName extends SuperclassName {

    // body

}

Variables and Methods

A subclass inherits all the member variables and methods from its superclass. It can add more unique variables and methods. It can also override methods of its superclass by providing its own implementation. However, static methods cannot be overridden, only hidden by declaring a method of the same name. It can make a superclass variable more or less visible by defining a variable of the same name and changing the access control modifier. 


The annotation "@Override" indicates to the compiler that the method below will override a superclass method. It is optional, but will make debugging and reading code easier. Since it is an annotation, it is discarded after compilation. 

Constructor

The subclass does not inherit the constructors of its superclasses. All constructors of a subclass must call a superclass constructor on its first line. If the superclass constructor is not called, the compiler will insert the call to the default constructor of the superclass in the first line. If the default constructor has not been created, all subclass constructors must call a superclass constructor on the first line. 

Keyword "super"

Similarly to "this", which is used to reference this instance, "super" is used to reference the superclass. The referenced superclass could be the immediate parent or its ancestor. 


Function

Usage

Call the superclass constructor

super(argumentList)

Call a superclass method

super.methodName(argumentList)

Reference a superclass variable

super.variableName

// Student.java

public class Student extends Person {

    // Makes the number variable of the Person class public

    public int number;


    // New variable belonging exclusively to the Student class

    private String name;


    public Student() {

        super(0); // Calls the super constructor to initialize the object

        name = "Unnamed Student";

    }


    // This constructor will cause the compiler to throw an error because the default constructor of the Person class is not defined

    public Student(String name) {

        this.name = name;

    }


    // Overrides the increase method of the Person class

    @Override

    public void increase(int n) {

        number -= n;

    }

}

UML Diagram

In UML Diagrams, inheritance is depicted with an arrow pointing from the subclass to the superclass. 


Common Root Class

All Java classes have a common root called java.lang.Object. This class defines and implements common behaviors that are needed for the objects to run in the Java Runtime Environment. They also allow features like garbage collection and multithreading. 

Single Inheritance

Multiple inheritance is not supported by Java. A subclass can only have one direct superclass. This is called single inheritance. The reason is that conflicting implementations for the same method will be difficult to resolve.  On the other hand, a superclass may have multiple subclasses. 

Polymorphism

A subclass inherits all the attributes and operations of the superclass. So, a subclass object can do everything that the superclass can do. When a superclass instance is expected, a subclass instance can be substituted in place of it, and everything will still work. This is polymorphism. 

Upcasting and Downcasting

When a subclass instance is declared as a superclass instance to the compiler, it’s called upcasting. Since a subclass possesses all the properties of its superclass, upcasting is always safe. So, upcasting can be inserted implicitly by the compiler if needed. 


Downcasting is when a superclass instance is casted to a subclass instance. It is not always safe because the instance may not belong to the correct subclass. So, casting needs to be explicitly declared. In addition, a runtime exception may occur if the instance does not belong to the subclass it’s being casted to. 

The "instanceof" Operator

A binary operator called "instanceof" may be used to check if an object instance of a particular class. 

Syntax: 

objectName instanceof className

Example

// Shape.java

public class Shape {


    private String color;


    public Shape(String color) { // constructor

        this.color = color;

    }


    // The area cannot be found if the shape is not known

    public double getArea() {

        System.out.println("Cannot calculate area without knowing the shape!");

        return 0.0; 

    }


    // In this case, the superclass can find the color because it is stored

    public String getColor() {

        return color;

    }

}


// Rectangle.java

public class Rectangle extends Shape {


    private double length;

    private double width;


    // Initializes the rectangle with dimensions

    public Rectangle(String color, double length, double width) {

        super(color);

        this.length = length;

        this.width = width;

    }


    // Overrides the getArea() method of the Shape class

    @Override 

    public double getArea() { // finds the area of the shape

        return length * width;

    }

}


// Circle.java

public class Circle extends Shape {


    private double radius;

    

    // Initializes the circle with a radius

    public Circle(String color, double radius) {

        super(color);

        this.radius = radius;

    }


    @Override

    public double getArea() { // finds the area of the shape

        return Math.PI * radius * radius;

    }


    // Overrides the getArea() method of the Shape class

    public double getCircumference() { // Another method to find the circumference

        return 2 * Math.PI * radius;

    }

}


// Main.java

Shape[] shapes = new Shape[3];


// These are allowed since both rectangles and circles are shapes

shapes[0] = new Rectangle("red", 5, 3);

shapes[1] = new Rectangle("blue", 2, 6);

shapes[2] = new Circle("yellow", 4);


// Cycles through all shapes in the shapes array

for (Shape shape : shapes) {

    System.out.println(shape.getArea()); // Always possible to find area of the shape

    if (shape instanceof Circle) { // Uses instanceof operator

        Circle circle = (Circle) shape; // Safe to downcast to a circle because of the instanceof check

        System.out.println(circle.getCircumference()); // Now that the circle is declared as a circle during compile time, the getCircumference() method can be called

    }

}

Abstract Classes

Sometimes, there may not be a suitable definition for a method of a class. Instead of using a definition that throws an error, the method can be defined as abstract. The keyword "abstract" is used to define an abstract method, which has no implementation. 


Abstract methods may only appear in abstract classes. An instance of an abstract class cannot be made, even if a constructor for the abstract class is used. 


All non-abstract classes that extend an abstract class must provide implementation for the abstract class. An abstract method should not be declared final because then it cannot be overridden. 

The shape class is an example of when the class can be declared abstract. 


Example: 

public abstract class Shape {


    private String color;


    // Abstract classes can have constructors although they cannot be initialized

    public Shape(String color) {

        this.color = color;

    }


    // An abstract method for subclasses to implement

    abstract public double getArea();


    // Abstract classes can still have implemented methods

    public String getColor() {

        return color;

    }

}

The Rectangle and Circle classes can remain unchanged. 

Interfaces

An interface is a completely abstract superclass. It defines a set of methods its subclasses must support. The keyword "interface" is used instead of "class" to define an interface. All the methods in an interface are public and abstract by default. Interfaces cannot have constructors. Like abstract classes, interfaces cannot be instantiated. 


As noted before, a subclass can only extend one superclass. However, a class can implement multiple interfaces. To implement an interface, the keyword implements is used instead of extends. If a class implements multiple interfaces, they are separated by a comma. However, if an interface is extended by another interface, the keyword extends is used. An interface, like a class, can still extend multiple interfaces. 


All fields inside an interface are implicitly and always public, static, and final. The inclusion of these keywords is optional. It is not possible to declare a field with another modifier. 


An interface is like a contract for what a class that implements it must do. It is mostly used over abstract classes when classes that implement it are not related to each other. Examples of interfaces in the Java library are Cloneable, Comparable, and Runnable. 


Example: 

// Shape.java

public interface Shape {

    double getArea(); // public and abstract modifiers are not needed

}


// Circle.java

public class Circle implements Shape {


    private double radius;


    // Creates circle with radius

    public Circle(double radius) {

        this.radius = radius;

    }


    @Override

    public double getArea() { // finds the area of the shape

        return Math.PI * radius * radius;

    }

}


JDK 8 introduced default and static methods in interfaces. Methods that are default have implementations provided in the body, so classes that implement them do not have to provide their own implementations. They are denoted with the default keyword. If a class implements two interfaces with the same default method, it must provide its own implementation of the method to avoid ambiguity. Static methods work the same way as static methods in a normal class. Their implementations must be provided. 


JDK 9 introduced private methods in interfaces. These methods work like private methods in a normal class. They may be static. However, an implementation must be provided because subclasses cannot access and therefore cannot override private methods. 

Encapsulation

Encapsulation refers to keeping some data and methods inside the class so that they cannot be accessed by other classes. In APCSA, all instance variables must be private. To access or change them, get and set methods must be used. The benefit is that the creator of the class has complete control over how the data is accessed and changed. 


Example: 

// Time.java

public class Time {

    private int hour;

    private int minute;

    private int second;

    private int secondsSinceMidnight;


    // Constructor that uses methods to initialize the variables (may throw IllegalArgumentException)

    public Time(int hour, int minute, int second) {

        setHour(hour);

        setMinute(minute);

        setSecond(second);

        setSecondsSinceMidnight();

    }


    // The user can always get the hour

    public int getHour() {

        return hour;

    }


    // There are restrictions on what the variable hour can be

    public void setHour(int hour) {

        if (hour < 0 || hour > 23) { // Checks whether the hour is valid

            throw new IllegalArgumentException("Hour must be between 0 and 23");

        }

        // If hour is valid, sets the hour and updates secondsSinceMidnight

        this.hour = hour;

        setSecondsSinceMidnight();

    }


    // The rest of these methods work the same way as the getHour() and setHour() methods


    public int getMinute() {

        return minute;

    }


    public void setMinute(int minute) {

        if (minute < 0 || minute > 59) {

            throw new IllegalArgumentException("Minute must be between 0 and 59");

        }

        this.minute = minute;

        setSecondsSinceMidnight();

    }


    public int getSecond() {

        return second;

    }


    public void setSecond(int second) {

        if (second < 0 || second > 59) {

            throw new IllegalArgumentException("Second must be between 0 and 59");

        }

        this.second = second;

        setSecondsSinceMidnight();

    }


    public int getSecondsSinceMidnight() {

        return secondsSinceMidnight;

    }


    // Updates the secondsSinceMidnight

    private void setSecondsSinceMidnight() {

        this.secondsSinceMidnight = hour * 3600 + minute * 60 + second;

    }


    // Converts the object to a String

    @Override

    public String toString() {

        return String.format("%02d:%02d:%02d", hour, minute, second);

    }

}


// Main.java

Time time = new Time(12, 34, 56);

System.out.println(time); // prints "12:34:56"


time.setHour(23);

time.setMinute(59);

time.setSecond(59);

System.out.println(time); // prints "23:59:59"


time.setHour(24); // throws IllegalArgumentException
